02 / Pain Points
核心痛点
品牌素材散落各处，内容创作效率低，多渠道分发困难
📋

素材分散无序

品牌文档、图片、视频散落在个人电脑和各类文件夹中，缺乏集中管理机制，找素材全靠记忆和微信传文件

⏲

内容创作低效

门店依赖总部提供脚本和 demo 视频，自主创作专业度不足，难以针对不同渠道快速产出差异化内容

🔓

合规风险

门店自创内容无法全覆盖审核，存在违反广告法表述风险，品牌调性难以统一保障

📈

多品牌管理难

龙膜、威固、圣科、龙小霞四个品牌调性各异，素材混用容易造成品牌形象混淆

05 / Module 1
智能素材库
AI 驱动的品牌数字资产管理平台
功能说明
多品牌资产库 按品牌独立分区，支持图片、视频、PDF、PPT、PSD 等全格式上传管理
AI 自动打标 上传即识别：品牌归属、产品系列（前挡/侧后/天窗）、适用车型、内容类型、适用渠道
智能搜索 自然语言搜索，如 "龙膜畅悦70 适合小红书的对比图"，支持相似图搜索
版本管理 同一素材多版本追踪，标记最新/归档/废弃，防止使用过期素材
权限管理 总部品牌经理 / 区域经理 / 门店 分级访问控制
使用统计 下载量、使用频次、热门素材排行，数据驱动内容策略
ZoeyTalk / Eastman AI Proposal
06 / Module 2
AI 内容工厂
选素材 + 选渠道，AI 自动出内容
功能说明
渠道适配生成 选定素材 + 目标渠道（抖音/小红书/微信），自动生成符合平台调性和尺寸的内容
文案生成 种草文、测评文、活动推广、朋友圈话术，支持专业/亲民/促销多风格
短视频脚本 输入产品 + 卖点，输出分镜脚本（画面描述 + 口播 + 字幕），适配 15s/30s/60s
图文排版 自动排版小红书笔记、公众号图文，支持品牌模板定制
批量变体 一键生成多个内容变体（不同角度/风格），方便 A/B 测试
合规检查 AI 自动检测广告法违规（绝对化用语、虚假宣传），标注风险并建议修改

07 / Architecture
技术架构
全国产技术栈，数据不出境，按量付费
┌───────────────────────────────────────────────┐ │ Web 管理后台 | 移动端 H5/小程序（二期） │ ← 用户层 └──────────────────────┬────────────────────────┘ │ ┌──────────────────────┴────────────────────────┐ │ 素材管理服务 | AI 内容生成 | 分发管理服务 │ ← 应用层 │ 品牌知识库 & 合规引擎 │ └──────────────────────┬────────────────────────┘ │ ┌──────────────────────┴────────────────────────┐ │ 多模态理解 | 向量检索(RAG) | 内容生成(LLM) │ ← AI 层 │ 通义千问 / DeepSeek / 文心一言 │ └──────────────────────┬────────────────────────┘ │ ┌──────────────────────┴────────────────────────┐ │ 对象存储(OSS) | 向量数据库 | 关系型DB │ ← 数据层 └───────────────────────────────────────────────┘

技术选型要点

  • AI 模型：优先国产大模型（通义千问 / DeepSeek / 文心），数据合规无出境风险
  • 向量检索：Embedding + RAG 实现语义搜索，支持文本+图片跨模态检索
  • 存储：阿里云 OSS，支持图片/视频/文档大容量存储
