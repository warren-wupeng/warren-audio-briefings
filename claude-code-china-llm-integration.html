01 / Background

背景与目标

Claude Code 是 Anthropic 官方 AI 编程 CLI 工具。在国内企业环境中，直连海外 API 存在网络不稳定、合规风险等问题。本方案调研国内大模型厂商的协议兼容性，为企业内网部署提供可行路径。

02 / Protocols

Claude Code 四种 API 协议

Claude Code 原生支持四种不同的 API 协议接入方式，覆盖主流云平台。

01

Anthropic Messages API

ANTHROPIC_BASE_URL

Anthropic 原生协议。国内兼容厂商可直接替换端点地址，最简单的接入方式。支持完整的 Messages API 功能集。

02

AWS Bedrock

CLAUDE_CODE_USE_BEDROCK=1

通过 AWS Bedrock 托管 Claude 模型。适合已有 AWS 基础设施的企业。需要 IAM 认证，支持 VPC 私有端点。

03

Google Vertex AI

CLAUDE_CODE_USE_VERTEX=1

通过 Google Cloud Vertex AI 访问 Claude。适合 GCP 环境企业。支持 Google IAM 和 VPC Service Controls。

04

Azure AI Foundry

CLAUDE_CODE_USE_FOUNDRY=1

通过 Microsoft Azure AI Foundry 接入。适合 Azure 环境企业。支持 Azure AD 认证和 Private Endpoints。

!
此外还支持两种代理模式：Corporate Proxy（HTTPS_PROXY）和 LLM Gateway（如 LiteLLM），可叠加使用。

07 / Configuration

快速配置示例

三行命令即可完成接入配置。

Direct: Alibaba Cloud Bailian
# 直连阿里云百炼 (推荐)
export ANTHROPIC_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode"
export ANTHROPIC_API_KEY="sk-your-key"
export ANTHROPIC_MODEL="qwen-max"
claude
Direct: DeepSeek
# 直连 DeepSeek (性价比高)
export ANTHROPIC_BASE_URL="https://api.deepseek.com"
export ANTHROPIC_API_KEY="sk-your-key"
export ANTHROPIC_MODEL="deepseek-chat"
claude
LiteLLM Gateway: config.yaml
model_list:
  # 主力: 阿里云
  - model_name: qwen-max
    litellm_params:
      model: qwen-max
      api_base: https://dashscope...

  # 代码: DeepSeek
  - model_name: deepseek-coder
    litellm_params:
      model: deepseek-coder
      api_base: https://api.deepseek...

router_settings:
  routing_strategy: least-busy
  fallback_models: [qwen-max, deepseek]

规模化部署
1-2 月
部署 LiteLLM 网关。
多厂商接入 + 负载均衡 + 故障切换。
统一密钥管理 + 成本追踪 + 审计日志。
Phase 3
私有化
3-6 月
内网部署 Qwen-72B 或 DeepSeek-V3。
敏感代码走内网模型，非敏感走外网。
满足数据安全合规要求。
Phase 1 推荐厂商
首选: 阿里云百炼 + Qwen — 企业级 SLA，模型最丰富
备选: DeepSeek — 代码能力突出，性价比极高
注意事项
API 兼容性需实测（tool_use / caching）
模型代码能力与 Claude 仍有差距
部分行业需等保认证厂商

09 / Summary

核心结论

1
国内厂商可直接接入 Claude Code

阿里云百炼、MiniMax、DeepSeek、ModelScope 原生支持 Anthropic Messages API，修改一个环境变量即可。

2
LiteLLM 网关覆盖所有国内厂商

OpenAI 兼容的厂商（豆包、文心、GLM、Kimi 等）通过 LiteLLM 协议转换后均可接入，覆盖率 100%。

3
Bedrock / Vertex / Foundry 无国内厂商支持

Claude Code 的三种云平台协议目前无国内对应方案。Anthropic Messages API 是唯一的直连路径。

4
企业级方案: LiteLLM 网关 + 阿里云百炼 + DeepSeek

推荐部署 LiteLLM 做统一入口，主力用阿里云百炼（企业 SLA），代码任务用 DeepSeek（性价比），自动故障切换。
