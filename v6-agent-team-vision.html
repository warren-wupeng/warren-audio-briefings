Three Agent Roles

PM Agent
Understands user intent. Breaks down requirements into Issues. Assigns, coordinates, and accepts deliverables. Makes product decisions.
Tools: create_issue, assign_task, check_progress, send_message, read_sop
Engineer Agent
Receives well-defined Issues. Writes code, runs tests, fixes bugs. Strictly executes within scope. Reports back on completion.
Tools: all 15 V5 tools + send_message, check_mailbox
Strategist Agent
Writes README, docs, CHANGELOG. Focuses on product narrative and user experience. Scans for improvement opportunities.
Tools: file R/W + send_message, check_mailbox, web_search

Core Mechanism: Heartbeat

Agents don't wait for instructions. They wake up, check state, and push forward.

def tick(self): # Every agent runs this on a configurable interval self.git_pull() # 1. Sync latest state self.check_mailbox() # 2. Read new messages self.check_tasks() # 3. Check assigned issues self.execute_sop() # 4. Follow role-specific SOP self.push_progress() # 5. Commit & push updates

The key phrase is "continue pushing forward" -- agents find work to do, not wait for it.

Git as Shared Brain

All agents share state through a single Git repository. Persistent, auditable, human-readable.
